相机

在阅读下述段落前，我们先回忆一下平时我们拍照的一个过程

  • 先找一个地方，把所有人都集中在这个地方中，这个过程就是模型变换（改变人的位置）

  • 把相机调整到一个好的位置，和一个好的角度，把相机放好，这个过程就是视图变换

  • 最后按下快门把相机中看到的场景投影到相片上，这个就是投影变换

模型变化

局部坐标系 -> 世界坐标系

视图矩阵

就是上面说的如何摆放相机

世界坐标系 -> 观察坐标系

也就是确定

  • 相机放置的位置 position

  • 相机该看向哪里 lookAt

  • 相机的向上方向（up）这里可以理解为相机怎么拿，横着拿还是竖着拿

通常，我们会把相机永远放在原点上（0，0，0），相机沿着-z 方向上看，向上方向是 y

position (0,0,0,1)
lookAt (0,0,-1)
up (0,1,0)

也就是解决如何将世界坐标中的点转到到相机这个坐标系下

有两种等价思维

  • 移动相机，将相机从当前位置移动并旋转到标准位置

  • 移动世界，将整个世界进行相反的移动和渲染，让相机相对处于标准位置

视图矩阵就是用的第二种思路，在数学上，这个其实就是相机自身变换的逆矩阵

构建视图矩阵

构建视图矩阵的关键在于，通过三个参数计算除相机局部坐标系的三个相互垂直的轴，并用这些轴构建一个旋转矩阵，再结合相机的位置平移，最终形成视图矩阵

计算相机的新的坐标系

  • Z 轴
    zAxis = normalize(position - target)

这里需要求的是 Z 轴的正方向，由目标点指向相机

  • X 轴（右向量）
    通过上方向和 Z 轴叉乘
    xAxis = normaliz(cross(up,zAxis))

  • Y 轴（实际上向量）
    通过 Z 轴和 X 轴叉乘得到，以确保三个轴相互垂直

yAxis = cross(zAxis, xAxis)

组合逆变换

正常我们会先平移再旋转

那么逆变化就会先旋转再平移

通过上面构建的旋转矩阵再乘上平移的逆矩阵

最终的结果就是
-position.dot(xAxis)
-position.dot(yAxis)
-position.dot(zAxis)

投影矩阵

观察坐标系 -> 裁剪坐标系（NDC 空间）

本质上还是坐标轴的转换

这里也就是把三维场景变成观察到的二维场景

正交相机

在三维空间中我们会规定一个能见区域

正交相机下

这个区域是一个立方体

这个立方体需要设置

宽度（right - left）

高度（top - bottom）

深度 （near - far）

如果我们要把它映射到 NDC 空间

NDC 空间就是一个
X,Y,Z 都在 (-1,1) 这个区间里面

就是把大立方体中心点移动到坐标原点，然后再进行缩放

本质上还是平移缩放

  • 平移

中心点移动到 0，0

((left + right) / 2)

  • 缩放

scale = 2 /（right - left）

如何得到

right - left = width

width * scale = (1 -(-1))

透视相机

透视相机会有近大远小的视觉效果，我们的转换思路是，想把这个四棱台也压缩成立方体，套用正交投影的过程

也就是我们其实只需要得到一个立方体转换为棱台的矩阵，乘一次就好了

俯仰角 Pitch

往上看或者往下看的角

固定 y 轴

方向分量变化

direction.y = sin(glm::radians(pitch)); direction.x = cos(glm::radians(pitch)); direction.z = cos(glm::radians(pitch));

偏航角 Yaw

固定 z 轴

往左往右看

基本和俯仰角一致

基于俯仰角和偏航角我们能得到

direction.x = cos(glm::radians(pitch)) * cos(glm::radians(yaw)); // 译注：direction代表摄像机的前轴(Front)，这个前轴是和本文第一幅图片的第二个摄像机的方向向量是相反的 direction.y = sin(glm::radians(pitch)); direction.z = cos(glm::radians(pitch)) * sin(glm::radians(yaw));
